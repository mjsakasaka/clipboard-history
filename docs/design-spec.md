# UI/UX 设计规范

## 设计原则

1. **简洁** — 减少视觉噪音，每个元素都有存在的理由
2. **直观** — 用户不需要说明书就能操作
3. **高效** — 最常用的操作（查找、粘贴）不超过 2 次点击
4. **内敛** — 软件平时不打扰用户，需要时才出现

---

## 交互规范

### 点击卡片 → 复制

1. 用户点击卡片内容区域
2. 内容写入剪贴板
3. 卡片短暂高亮（背景变 `#E8EAF6`，200ms）
4. 窗口底部弹出 Toast："已复制到剪贴板"（1.5 秒后自动消失）

### 置顶/取消置顶

1. 点击 📌 按钮
2. 立即切换状态（无动画）
3. 卡片移动到列表顶部（置顶）或回到时间排序位置（取消）

### 删除

1. 点击 🗑 按钮
2. 弹出确认对话框："确定要删除这条记录吗？"
3. 确认后卡片消失，数据库记录和关联图片文件被删除

### 搜索

1. 在搜索框输入关键词
2. 列表实时过滤，仅显示匹配的文字记录
3. 清空搜索框恢复完整列表
4. 无匹配结果时显示空状态："没有找到相关内容"

## 空状态

当没有任何历史记录时，主窗口显示：

```
┌──────────────────────────────────┐
│                                  │
│         📋 (灰色大图标)           │
│                                  │
│      还没有复制任何内容           │
│   试试复制一段文字或一张图片吧     │
│                                  │
└──────────────────────────────────┘
```

- 图标颜色：`#BDBDBD`
- 文字颜色：`#9E9E9E`
- 图标大小：48x48px

## Toast 提示

```
│  ✅ 已复制到剪贴板         │
└──────────────────────────┘
```

- 位置：窗口底部居中
- 背景：`#424242`（深灰）
- 文字：`#FFFFFF`
- 圆角：4px
- 持续时间：1.5 秒
- 淡入淡出动画：300ms
